# Release Contract

## Суть

Проданные Carrd-шаблоны получают theme и plugin assets только как inline embeds, вставленные напрямую в Carrd (`Hidden → Head` / `Hidden → Body End`). Никакой CDN/jsDelivr-доставки не существует: `dist/` не содержит `*.min.css`/`*.min.js` бандлов или `*-cdn.html` снипетов, предназначенных для внешней ссылки.

`main` используется для разработки и проверки. Он не является каналом доставки покупателям.

## Версионирование

Проект использует нестандартную (для SemVer) нумерацию, зафиксированную решением владельца:

- `2.0.0` — переход на inline-embed-only доставку (эта версия). Ниже по номеру, чем предыдущий `2.1.0`, но это осознанно отдельная линия, а не продолжение `2.1.x`.
- `2.1.0` и более старые теги остаются в git-истории навсегда и продолжают отдавать файлы уже опубликованным `@2.1.0`-сайтам через jsDelivr — они просто больше не активный путь доставки для новой работы.
- `3.0.0` зарезервирован на будущее: если CDN/jsDelivr-доставка когда-либо будет возвращена, это будет `3.0.0`, а не патч/минор к `2.0.x`.
- Внутри `2.0.x`: patch — исправление без изменения публичной разметки/embed-контракта; minor (`2.x.0`) — новая функциональность без нарушения embed-контракта; следующий major (`3.0.0`) — зарезервирован под возврат CDN, либо под любое другое несовместимое изменение embed-контракта, если оно случится раньше.
- Номер в `VERSION` и changelog должны совпадать. Git tag `v<version>` создаётся при публикации.
- Повторно создавать, перемещать или force-push существующий release tag запрещено (включая старые `v2.1.0` и более ранние).

## Каналы доставки

- `main` — разработка, review, live-доработка draft.
- `vX.Y.Z` — immutable Git tag, отмечающий состояние репозитория на момент релиза (для истории/провенанса, не для CDN-доставки).
- Inline embed — единственный канал доставки покупателю: файлы `dist/*-embed.html`, `dist/*-embed-part{1,2}.html`, `dist/theme-design-system.html`, `dist/theme-design-tokens-embed.html`, `dist/theme-ui-embed.html` вставляются в Carrd как есть.
- CDN/jsDelivr — не используется. Если он понадобится в будущем, это отдельное решение под номером `3.0.0`, требующее отдельного пересмотра этого контракта.

## Что происходит со старыми версиями

Старые версии не исчезают. Сайты, ранее подключённые через `@2.1.0` CDN, продолжают получать файлы из тега `v2.1.0` через jsDelivr неограниченно долго — эти теги не удаляются и не перемещаются. Но это не активный путь для новых поставок: любой новый сайт или обновление существующего использует inline embeds из текущего `dist/`.

## Детали

### Когда выпускать версию

- Patch (`2.0.x+1`): локальное исправление без изменения embed-разметки, `data-*` контракта или настроек.
- Minor (`2.x+1.0`): новая возможность или заметное изменение поведения при сохранении обратной совместимости embed-контракта.
- Major (`3.0.0`): возврат CDN-доставки, либо удаление/переименование публичного embed-контракта, которое нельзя выполнить совместимо.

### Release process

1. Подготовить изменения в `main`.
   - Изменить только `src/`, source README, config и внутреннюю документацию.
   - Записать изменения в `CHANGELOG.md` под `[Unreleased]`.
   - Не редактировать `dist/` вручную — только через `npm run build`.

2. Назначить версию.
   - Выбрать patch или minor по правилам выше (major зарезервирован под возврат CDN или другое несовместимое изменение).
   - Записать номер без `v` в `VERSION`.
   - Перенести готовые записи из `[Unreleased]` в секцию `[X.Y.Z] - YYYY-MM-DD`.

3. Подготовить release candidate.

   ```bash
   npm run release:prepare
   ```

   Команда пересобирает `dist/`, запускает полный `validate` и затем `release:check`. Preflight проверяет SemVer-формат, наличие датированной секции changelog и отсутствие уже существующего tag. После неё проверить diff и убедиться, что рабочее дерево содержит только изменения этого релиза.

4. Зафиксировать immutable release.

   ```bash
   git add <release-files>
   git commit -m "Release: vX.Y.Z"
   git tag -a vX.Y.Z -m "Release vX.Y.Z"
   git push origin main
   git push origin vX.Y.Z
   ```

5. Обновить Carrd.
   - Открыть каждый изменившийся `dist/*-embed.html` и вставить актуальный текст в соответствующий Carrd embed (заменить целиком, не патчить построчно).
   - Проверить Builder draft, desktop и mobile.
   - Опубликовать Carrd только после ручного подтверждения владельца.
   - Выполнить post-publish smoke.

### Pre-sale validation

Перед тем как считать шаблон готовым к продаже, обязательно проверить:

- В Carrd нет ссылок на `cdn.jsdelivr.net` — это признак незавершённой миграции со старой CDN-доставки.
- Каждый установленный `Head`/`Body End` embed совпадает по содержимому с актуальным `dist/*-embed.html` на момент передачи.
- Published page реально отражает вставленный embed, а не устаревшую вставку.

### Продажа шаблона

- Каждый передаваемый покупателю шаблон получает файлы только как inline embeds — без CDN-ссылок любого вида.
- Site-owned embeds с tokens, custom CSS, `window.CarrdPluginOptions` и custom JS остаются внутри Carrd и не входят в `dist/`.
- Покупатель не получает обновление автоматически. Новая версия — это новый пакет embed-файлов, который нужно вставить заново вручную, с перечнем изменений и, при необходимости, инструкцией миграции.

### Hotfix

Если ошибка найдена после продажи: исправить в `main`, выпустить следующий patch, подтвердить на canonical smoke surface, затем вручную заменить embed-содержимое на выбранных сайтах.

### Запрещено

- Ссылаться на `cdn.jsdelivr.net` в любом виде в продаваемом шаблоне.
- Перемещать `vX.Y.Z` на новый commit или удалять опубликованный tag (включая старые `v2.1.0` и более ранние — они остаются провенансом для сайтов, всё ещё живущих на старой CDN-доставке).
- Публиковать изменённый embed-контент под уже использованным номером версии.
- Обещать покупателю авто-обновление без ручной переустановки embed.

### Готово

- `VERSION`, changelog и tag совпадают.
- `npm run release:prepare` завершён без ошибок, включая `release:check`.
- Release commit и annotated tag отправлены в GitHub.
- Carrd использует свежий embed-контент, не содержит ссылок на `cdn.jsdelivr.net`, прошёл post-publish smoke.
- Старые tags сохранены без изменений.
